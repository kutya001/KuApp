## 1. Обзор стратегии тестирования

### 1.1 Подход

KuApp — это zero-build проект без npm-зависимостей. Тестирование проводится на двух уровнях:

| Уровень | Метод | Инструменты |
|---------|-------|-------------|
| **Unit-тесты** | Отдельные JS-функции (утилиты, валидация) | Встроенные `console.assert` или простой test runner |
| **Интеграционные тесты** | Взаимодействие с Google Sheets через GAS | Ручное тестирование + скрипты |
| **E2E-тесты** | Полный пользовательский сценарий | Ручное тестирование в браузере |
| **PWA-тесты** | Service Worker, кэширование, установка | Chrome DevTools → Application |

### 1.2 Окружения

| Окружение | URL | Назначение |
|-----------|-----|-----------|
| **Dev** | `http://localhost:5500` (Live Server) | Локальная разработка |
| **Staging** | `https://kutya001.github.io/KuApp/` (main branch) | Финальная проверка перед релизом |
| **Production** | Тот же URL | Боевое окружение |

### 1.3 Браузеры для тестирования

| Браузер | Версия | Приоритет |
|---------|--------|-----------|
| Chrome (Desktop) | Latest | P0 |
| Firefox (Desktop) | Latest | P1 |
| Safari (Desktop) | Latest | P1 |
| Chrome (Mobile Android) | Latest | P0 |
| Safari (iOS) | Latest | P1 |
