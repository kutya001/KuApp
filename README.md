## 1. Обзор проекта

**McGill** — мобильное прогрессивное веб-приложение с тремя независимыми модулями:

| Модуль | Описание |
|---|---|
| 🏋️ Фитнес-тренер | Таймер с голосовыми подсказками для реабилитационных и HIIT-программ |
| 📖 Словарь | Изучение иностранных слов через карточки и тесты с бэкендом на Google Sheets |
| 🎮 Мини-игры | Коллекция браузерных игр (Тетрис, Змейка, 2048, Судоку и другие) |

Всё работает **без сервера** — чистый HTML/CSS/JS, хостинг на GitHub Pages.

## 3. Модуль: Фитнес-тренер

Запускается с главной страницы (`index.html`). Показывает пошаговый протокол тренировки с таймером и голосовыми подсказками.

### Тренировочные программы

| Программа | Файл | Время | Описание |
|---|---|---|---|
| Большая тройка МакГилла | `mcgill_531.json` | ~15 мин | Реабилитация позвоночника: Bird-Dog, Side Bridge, Curl-Up |
| Реабилитация плеча — Фаза 1 | `shoulder_rehab_p1.json` | ~7 мин | 0–4 недели после операции: защита, снятие боли |
| Реабилитация плеча — Фаза 2 | `shoulder_rehab_p2.json` | — | 4–8 недель: восстановление диапазона движения |
| Реабилитация плеча — Фаза 3 | `shoulder_rehab_p3.json` | — | 8–12 недель: укрепление ротаторной манжеты |
| Реабилитация плеча — Фаза 4 | `shoulder_rehab_p4.json` | — | 12+ недель: возврат к активности |
| Домашняя HIIT | `home_hiit_fatloss_001.json` | — | Сжигание жира, без оборудования, 3 круга |

**Управление тренировкой:**
- Выберите программу из выпадающего списка
- Нажмите ▶ для старта / ⏸ для паузы
- Голосовые подсказки произносятся автоматически через Web Speech API
- Цвета прогресс-бара: 🟢 зелёный — работа, 🟠 оранжевый — отдых, 🔵 синий — подготовка

### Как добавить свою программу

**Шаг 1.** Создайте файл `workouts/my_program.json` по шаблону:

```json
{
  "programs": [
    {
      "id": "unique_id",
      "name": "Название программы",
      "description": "Описание",
      "estimatedTime": "20 минут",
      "exercisesDetails": {
        "Название упражнения": {
          "goal": "Цель упражнения",
          "start": "Исходное положение",
          "move": "Техника выполнения"
        }
      },
      "protocol": [
        {
          "ex": "Название упражнения",
          "phase": "Подготовка",
          "type": "prep",
          "time": 10,
          "voiceTxt": "Текст голосовой подсказки",
          "toneType": "prep",
          "stepDesc": "Описание шага для экрана"
        },
        {
          "ex": "Название упражнения",
          "phase": "Подход 1",
          "type": "work",
          "time": 30,
          "voiceTxt": "Начали",
          "toneType": "work",
          "stepDesc": "Выполнение упражнения 30 секунд"
        }
      ]
    }
  ]
}
```

**Типы шагов (`type`):**
- `prep` — подготовительный (синий)
- `work` — рабочий (зелёный)
- `rest` — отдых (оранжевый)
- `finish` — завершение (золотой)

**Шаг 2.** Добавьте запись в `workouts/index.json`:

```json
{
  "id": "unique_id",
  "title": "Название в меню",
  "file": "my_program.json",
  "category": "hiit",
  "description": "Краткое описание"
}
```

**Категории:** `rehab` — реабилитация, `hiit` — интенсивные, `strength` — силовые.

## 4. Модуль: Словарь (WordLearning)

**Путь:** `games/WordLearning.html`  
**Прямая ссылка:** https://kutya001.github.io/McGill/games/WordLearning.html

Приложение для изучения иностранных слов. Данные хранятся в Google Таблице — можно редактировать с любого устройства.

### Первоначальная настройка

При первом запуске потребуется:
1. **ID Google Таблицы** — берётся из URL таблицы
2. **JSON сервисного аккаунта** — файл с ключами для доступа к API

После ввода этих данных нажмите **«Подключиться»**. Если таблица новая и пустая — нажмите **«🗃 Инициализировать таблицы»** (или это произойдёт автоматически при первой синхронизации).

> **Автоматическая инициализация:** при каждой синхронизации приложение проверяет наличие всех нужных листов. Если каких-то не хватает или таблица пустая — листы и заголовки создаются автоматически.

### Настройка Google Sheets

**1. Создайте Google Таблицу**

Откройте [Google Sheets](https://sheets.google.com) и создайте новую таблицу. ID таблицы — часть URL:
```
https://docs.google.com/spreadsheets/d/ВОТ_ЭТОТ_ID/edit
```

**2. Создайте сервисный аккаунт**

1. Откройте [Google Cloud Console](https://console.cloud.google.com)
2. Создайте проект (или выберите существующий)
3. Включите **Google Sheets API**: *APIs & Services → Library → Google Sheets API*
4. Создайте сервисный аккаунт: *IAM & Admin → Service Accounts → Create Service Account*
5. Создайте JSON-ключ: *Service Account → Keys → Add Key → JSON*
6. Скачайте файл — это и есть «JSON сервисного аккаунта»

**3. Откройте доступ к таблице**

В таблице нажмите **«Поделиться»** и добавьте email сервисного аккаунта (вида `...@...iam.gserviceaccount.com`) с ролью **«Редактор»**.

**4. Инициализируйте таблицу**

Вставьте ID таблицы и JSON в форму настройки, нажмите **«🗃 Инициализировать таблицы»**. Приложение создаст 5 листов с правильными заголовками и добавит пользователя-администратора по умолчанию.

### Функции приложения

#### 📚 Главная (Словарь)
- Отображает слова по категориям с прогрессом изучения
- Выбор языковой пары (от → до) с кнопкой смены направления
- Поиск по словам
- Цветовые уровни: ⚫ не изучено → 🟠 учу → 🟡 знаю → 🟢 выучено

#### 🎴 Карточки (Флеш-карточки)
- Карточки отсортированы: новые → учу → выученные
- Переворот по нажатию или тапу
- Свайп вправо — знаю ✓, свайп влево — учить ✗
- Режим «Обратный» — угадываешь слово по переводу
- Прогресс сохраняется локально (localStorage)

#### ❓ Тест (Квиз)
- 10 вопросов, 4 варианта ответа
- После каждого ответа — подсветка правильного
- По результатам теста автоматически обновляется рейтинг

#### 🏆 Рейтинг
- Таблица лидеров по количеству выученных слов
- Показывает: позицию, логин, число слов, процент правильных ответов, дату активности
- Обновляется автоматически после каждого теста

#### ⚙️ Настройки
Шесть вкладок:

| Вкладка | Содержимое |
|---|---|
| Общие | Данные подключения, статистика кэша, сброс прогресса |
| Языки | Список языков с редактированием + добавление |
| Категории | Список категорий с иконками + редактирование |
| Данные | Массовый импорт слов в текстовом формате |
| Импорт/Экспорт | Умный JSON-импорт и экспорт по языковой паре |
| Админ | Управление пользователями *(только для администраторов)* |

### Система пользователей

#### Пользователь по умолчанию
При инициализации создаётся:
- **Логин:** `Kutman`
- **Пароль:** `123`
- **Роль:** администратор

> ⚠️ Смените пароль после первого входа.

#### Роли
| Роль | Возможности |
|---|---|
| `user` | Изучение слов, карточки, тесты, рейтинг |
| `admin` | Всё вышеперечисленное + CRUD языков, категорий, слов, управление пользователями |

#### Регистрация
1. На экране входа нажмите **«Регистрация»**
2. Введите логин и пароль (дважды)
3. Аккаунт создаётся со статусом **«ожидает подтверждения»**
4. Администратор одобряет регистрацию во вкладке **Настройки → Админ**

#### Смена пароля
Нажмите на имя пользователя в шапке → **«Сменить пароль»**.

### CRUD-редактирование данных

Все изменения немедленно сохраняются в Google Таблицу.

**Языки** (Настройки → Языки):
- Добавить: кнопка `+ Добавить язык`
- Редактировать / удалить: кнопка ✏ рядом с языком

**Категории** (Настройки → Категории):
- Добавить: кнопка `+ Добавить категорию`
- Редактировать / удалить: кнопка ✏ рядом с категорией

**Слова** (вид «Слова»):
- Добавить: кнопка `+` (FAB, правый нижний угол)
- Редактировать / удалить: кнопка ✏ на карточке слова

### Импорт и экспорт JSON

#### Экспорт
Настройки → Импорт/Экспорт:
1. Выберите языковую пару и (опционально) категорию
2. **«Скачать JSON»** — сохранит `.json`-файл на устройство
3. **«📋 Копировать в буфер»** — скопирует JSON в буфер обмена

Формат экспортируемого файла:
```json
{
  "langFrom": "ru",
  "langTo": "en",
  "category": "food",
  "exported": "2026-01-15T12:00:00.000Z",
  "words": [
    { "category": "food", "wordFrom": "Яблоко", "wordTo": "Apple", "transcription": "ˈæpəl" }
  ]
}
```

#### Импорт
1. Настройки → Импорт/Экспорт → **«Выбрать файл .json»**
2. **Умный импорт:**
   - Новые слова — добавляются
   - Существующие слова с изменёнными переводами — обновляются
   - Точные дубликаты — пропускаются
3. Тост покажет итог: `+5 новых, ↻2 обновлено, =1 пропущено`

#### Массовый текстовый импорт
Настройки → Данные:
- Выберите категорию
- Введите слова в формате `слово|перевод|транскрипция` (по одному на строку)
- Разделитель: `|` или Tab
- Поле транскрипции необязательно

```
apple|яблоко|ˈæpəl
banana|банан|bəˈnænə
cat|кошка
```

## 6. PWA и офлайн-режим

Приложение работает как **Progressive Web App**:

- **Установка:** на Android/iOS нажмите «Добавить на главный экран» в браузере
- **Офлайн:** главная страница и все игры кэшируются Service Worker'ом (`sw.js`)
- **Словарь:** без интернета работает с закэшированными данными (последняя синхронизация)
- **Версия кэша:** `app-v3` (обновляется при деплое)

**Стратегия кэширования:**
- Оболочка приложения (HTML, иконки, манифест) — Cache First
- Внешние API (Google Sheets) — Network Only (SW не перехватывает)

При потере связи в шапке появляется бейдж **OFFLINE**.

## 7. Технический стек

| Компонент | Технология |
|---|---|
| Frontend | Vanilla HTML5 / CSS3 / JavaScript (ES2022) |
| Шрифты | Google Fonts: Manrope, JetBrains Mono |
| API авторизация | Google Service Account JWT (RS256, Web Crypto API) |
| Хранилище данных | Google Sheets API v4 |
| Локальный кэш | `localStorage` |
| TTS (озвучка) | Web Speech API (`SpeechSynthesis`) |
| PWA | Service Worker + Web App Manifest |
| Хостинг | GitHub Pages |

**localStorage ключи (Словарь):**

| Ключ | Содержимое |
|---|---|
| `wl_config` | ID таблицы, выбранные языки |
| `wl_sa` | JSON сервисного аккаунта |
| `wl_cache` | Локальная копия всех данных (языки, категории, слова, пользователи, рейтинги) |
| `wl_progress` | Прогресс изучения слов (уровни по каждому слову) |
| `wl_user` | Текущий авторизованный пользователь |

## 8. Деплой на GitHub Pages

Репозиторий: https://github.com/kutya001/McGill

**Обновить деплой:**
```bash
git add -A
git commit -m "описание изменений"
git push origin main
```

GitHub Pages автоматически публикует изменения из ветки `main` в течение 1–2 минут.

**Обновить кэш Service Worker:**  
Если нужно, чтобы пользователи получили новую версию — смените константу в `sw.js`:
```js
const CACHE_NAME = 'app-v4';   // было app-v3
```
